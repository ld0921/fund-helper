## 🎯 核心发现：V2 推翻了 V1 的两个关键结论

### 发现 1：Phase 空头判断命中率 **54.5%**（优于随机），V1 的 44.4% 是数据偏差造成的假象

| 版本 | 多头命中率 | 空头命中率 |
|---|---|---|
| V1（类别预聚合数据） | 73.3% (15 次) | **44.4%** (9 次) |
| V2（日净值还原） | 57.1% (7 次) | **54.5%** (11 次) |

**含义**：
- findings.md P0 的 "overheat/stagflation 触发条件有缺陷" 这个结论**证据不足**
- V1 的数据本身有问题（可能是时间错位+幸存者偏差），导致 phase 看起来命中率低
- V2 真实数据下，phase 判断有**微弱但真实的 alpha**（空头 54.5% > 50%）

### 发现 2：V1 的收益/波动/回撤数字**严重失真**

| 指标 | V1 进取+2年 | V2 进取+2年 |
|---|---|---|
| 复利累计 | 89.06% | **48.39%** |
| 年化 | 37.50% | 24.02% |
| 年化波动 | **42.11%** | **14.75%** ⭐ |
| 最大回撤 | **33.94%** | **5.71%** ⭐ |
| 夏普 | 0.85 | **1.51** ⭐ |
| Sortino | 1.51 | **4.11** ⭐ |

**V1 波动率虚高 3 倍**：因为 V1 的 `marketBenchmarks.monthlyReturns` 有 +78.96%、-50.67% 等极端值（时间错位造成），V2 用真实日净值计算出的月度波动只有 3-5%，属于正常范围。

## ⚠️ V1 回测的数据质量问题（追溯）

看 `data/curated-details.json` → `marketBenchmarks.active.monthlyReturns`：
```
[-3.09, 25.34, -2.37, ..., 78.96, 10.78, -50.67, ...]
```

连续的 +78.96% / -50.67% 在真实基金类别平均月度收益里是**几乎不可能**的。V2 用日净值重算，相邻月份的类别平均月度收益都在 ±10% 以内。

**根因**（推测）：`scripts/fetch-ranks.js` 生成 `monthlyReturns` 时是按"数组索引"平均各基金的月度收益，但不同基金的成立日期不同，`monthlyReturns[0]` 对同一索引指向的是不同月份，导致聚合失真。

**影响**：V1 回测里所有涉及 `marketBenchmarks.monthlyReturns` 的逻辑都被污染，包括 `computeWeights` 的动态相关性矩阵、risk parity 标准差估算。

## 🔍 V2 下的新观察

### O1：算法仍然跑输基线，但夏普几乎持平

- 等权基线夏普 1.55，算法最佳 1.51 — 差距从 V1 的 0.03 变 0.04，基本持平
- 累计收益等权 62.75% 领先算法最佳 14pp — 和 V1 相似

### O2：Phase 分布完全不同

- V1（失真数据）：recovery:12, stagflation:5, overheat:4, global_bull:3
- V2（真实数据）：stagflation 占前 10 月，qdii_opp 过渡，recovery 后期 12 月

V2 正确识别了 **2024-05 ~ 2025-02 的 A 股弱势期**，这是 V1 由于幸存者偏差完全遗漏的重要事实。

### O3：回撤控制是算法的真价值

- V2 算法最大回撤 **5.2-5.9%** vs 基线 6.3-7.8%
- 算法用 ~1pp 的回撤改善换了 ~14pp 的累计收益损失
- **对回撤敏感的用户（大多数散户），算法仍有价值**
