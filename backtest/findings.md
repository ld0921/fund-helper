## 2. 关键发现（Findings）

### 🔴 F1 [P0] Phase 空头判断命中率 44%，低于随机

**数据**：
- 多头判断（`recovery` + `global_bull`）：15 次，命中率 **73.3%**（预测后月权益方向正确）
- 空头判断（`overheat` + `stagflation`）：9 次，命中率 **44.4%**
- 空头判断命中率 < 随机基线（50%），说明当前阈值在这段数据里是**反向信号**

**根因假设**：
- `overheat` 触发条件 `equitySig > 10 && bondWeak`：在持续牛市中触发后市场往往继续上涨
- `stagflation` 触发条件 `equityWeak && bondWeak`：可能把**短期回调**误判为**系统性弱势**

**优化建议**：
1. 提高 `overheat` 触发阈值：`equitySig > 10` → `equitySig > 15` 或 `> bondStd × 2.5`
2. `stagflation` 增加确认维度：不能只看 `equityWeak && bondWeak`，建议加上"过去 3 个月累计权益跌 > 10%"的趋势条件
3. 把 `overheat` 的 `equityMult` 从 0.85 调至 0.92（降低错判时的伤害）
4. `stagflation` 的 `equityMult` 从 0.85 调至 0.90
5. 考虑把 `stagflation` 的 `money` 强制下限 15% 降至 8%（在数据样本中未被证明有效）

---

### 🔴 F2 [P0] 算法跑输等权基线 10-15pp，Risk Parity 权重过度稀释到债券/货币

**数据**：
- 所有算法画像累计 58-89%，等权 100.86%
- 主要差距来源：**Risk Parity 把 30-45% 权重分给债券/货币**，在权益单边上涨市场中稀释收益

**根因假设**：
- `computeWeights` 以 `1/σ` 为初始权重，债券 σ≈2%，权益 σ≈15-20%，初始权重差 8-10 倍
- 风险偏好倾斜和期限倾斜可以缓解但无法逆转
- 平衡档默认 40% 权益下限已经是最高的 balanced 下限，但仍比等权 50%+ 权益的配比偏低

**优化建议**：
1. **重新审视平衡档权益下限**：当前 40% → 考虑 50%（需要权衡熊市防御能力，需要更长数据段验证）
2. **评估是否把 risk_free_rate 纳入 Risk Parity**：当前 bond σ 太低导致过度配置，可考虑给 bond σ 加一个 floor（比如 2%）使得 RP 不至于过度偏债
3. **"动量倾斜" 的幅度上限 (`maxTilt`) 偏小**：balanced=10%, aggressive=12% → 在明确牛市里把权益再加 15-20% 才能接近等权/60/40

### 🟡 F3 [P1] 反转保护 `avgR1 > 30` 机制在持续牛市中过早降权益

**代码位置**：`portfolio.js:627-631`

```js
if(topCatData.avgR1 > 40){
  effectiveTilt = -Math.round(maxTilt * 0.3);  // 反转为负
} else if(topCatData.avgR1 > 30){
  effectiveTilt = 0;  // 不倾斜
}
```

**问题**：
- 本次回测里权益类 `avgR1` 多次超过 30-40%（qdii 达 60%+）
- 算法在这些高点反而降低或不加仓权益
- **但后续月份权益继续上涨**，损失 alpha

**优化建议**：
1. 不能单看 `avgR1`，应结合**趋势持续性**：比如要求 `r3Ann > r1`（长期年化已追平短期）时才触发反转保护
2. 或者保留反转保护，但把阈值从 30/40 提高到 40/55
3. 或者反转保护只作用于 **"已持仓中已盈利 X%" 的具体基金**，不作用于类别层配置

### 🟡 F4 [P1] 永久 recovery 基线夏普 0.86，高于所有切换画像

**数据**：永久 recovery（固定 `equityMult=1.10, bondMult=0.93`） 夏普 0.86，所有切换画像夏普 0.83-0.85

**含义**：
- 在这段数据里，"一直轻度看多" 比"动态切换 phase" 更优
- Phase 切换本身引入了切换成本（判断错误的回撤），净价值为负

**优化建议**：
1. **提高切换的置信度要求**：当前 `spread > bondStd × 0.5` 就切换，改为 `spread > bondStd × 1.0`
2. **延长确认期**：当前"连续 2 次"改为"连续 3 次"
3. **把 recovery 设为默认 phase**：在没有明确 `overheat` / `stagflation` 信号时，默认给 `equityMult=1.05`（轻度看多）而非中性的 1.0
4. 或者简化：把 phase 从 7 种减到 3 种（中性 / 强势 / 弱势），降低切换错误率

---

### 🟡 F5 [P1] 短期期限（horizon=1）equityCap=30 过严

**数据**：
- 平衡+1年 累计 58%
- 平衡+2年 累计 68%
- 年化差约 **10pp**

**问题**：
- `equityCap = 30` 假设 1 年内有显著波动风险
- 但本数据段是持续牛市，30% 权益上限让 1 年期用户大幅跑输
- 熊市数据缺失，无法验证 30% 的防御价值

**优化建议**：
1. 在文档（`showParamHelp` 或 `index.html`）里**显式告诉用户**：1 年期是防御性配置，牛市可能错过大部分涨幅
2. 或者让 `equityCap` 也受 `phase` 影响：`recovery` phase 下 1 年期 cap 放宽到 40%
3. 或者引入"投资期限 + 预期市场状态"的联合维度，给用户更细粒度选项

---

### 🟢 F6 [Positive] 月度胜率最高 75%，用户体验稳定

**数据**：进取+2年月度胜率 75%，其他画像 66.7-70.8%

**含义**：
- 3/4 月份盈利，用户心理承受更容易
- 减少"看到亏损就赎回"的行为损失
- 相比基线（66.7-70.8%）在这点上算法有优势

---

### 🟢 F7 [Positive] 风险调整后（夏普/Sortino）不输基线

**数据**：进取+2年夏普 0.85 vs 等权 0.82，60/40 0.79

**含义**：
- 算法通过降低波动和回撤，换来了与基线相当的风险调整收益
- 对于**风险厌恶型用户**（大部分基民）这是有价值的
- 项目 README / 介绍页可以强调"风险调整后表现"而非"绝对收益"

## 4. 局限性声明

### 4.1 必须在行动前知道的限制

1. **样本量严重不足**：24 个有效月，任何单一参数调整的证据力度都较弱；连续 3-5 个月的噪声就可能反转结论
2. **数据仅覆盖牛市段**：约 2024-04 ~ 2026-03，**不含熊市/震荡市**
   - 算法的"防御逻辑"（phase 降权益、Risk Parity 偏债）**无法在本样本中被证伪**
   - 在熊市里可能价值很大，本回测看不到
3. **基准数据对齐偏差**：`marketBenchmarks.monthlyReturns` 是各基金月度收益按索引求平均，不同成立日期的基金时间位置会错位
   - 解释了为什么 active 类月度收益出现 +78.96% 等极端值
4. **类别层回测，不能拆分 alpha 来源**：`monthlyReturns` 是"类别 top 基金平均"，已经隐式包含选股 alpha，本回测无法拆分
5. **无交易成本**：类别层月度 rebalance 不计申赎费/滑点，实际会降低 2-5pp 累计收益

### 4.2 本回测**不能**回答的问题

- `scoreF` 选基算法好不好
- `selectFunds` 的 Top-K + 去重逻辑有效吗
- 换仓成本系数 0.0015 是不是偏乐观（用户原问题）
- 持仓诊断的 7 种信号谁更有用
- 熊市防御能力

### 4.3 要得到这些答案，需要什么（即方案 B 路径）

1. 拉每只基金的历史日/月净值（天天基金 JSONP 可拉）
2. 用当时全市场基金池（避免幸存者偏差），每月末重算 r1/r3/maxDD
3. 模拟完整流程：scoreF → selectFunds → computeWeights → 按 t+1 日净值成交
4. 覆盖 5+ 年（含 2022 熊市）

预估：方案 B 需要 2-4 周业余时间。
