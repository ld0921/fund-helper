## 💡 关键洞察

### 洞察 1：scoreF 是"粗分"信号，不是"精算"工具

- R² 只有 0.02-0.09（极低）— 意味着 scoreF 只能解释后续收益方差的 2-9%
- 但三分位 Top-Bot 差 20-30pp — 说明 scoreF 能**区分头尾**
- 使用方式：**作为筛选工具**而非**精确预测工具**

### 洞察 2：bond 和 qdii 的 scoreF 是噪声

- bond 类 R² 0.002 · Top-Bot 差 -1.4pp（甚至反向）
- qdii 类 R² 0.003 · Top-Bot 差 1.9pp
- **原因**：
  - bond 基金回报主要由利率和信用利差决定，scoreF 的 Calmar/一致性等指标对这些"宏观驱动"回报预测力弱
  - qdii 基金受海外市场影响大，国内数据的 scoreF 指标迁移性差

### 洞察 3：系数 0.0015 不需要改

- 全样本真实斜率 0.134%/分，代码假设 0.15%/分，**相差 10%** 属于采样噪声范围
- 精确到 0.0013 更准确，但改动价值小（1-2%）

---

## 🛠 建议的代码改动（非 P0）

### Option A：按类别差异化系数（推荐）

`calculateRebalanceCost` 对 bond/qdii 把 expectedGainRate 降为小系数或 0：

```js
const coefByCategory = {
  active: 0.007,  // 真实 0.0069
  index:  0.009,  // 真实 0.0091
  bond:   0.001,  // 真实 -0.0004，保守设为 0.001
  qdii:   0.001,  // 真实 0.0009
};
const coef = coefByCategory[targetFund.cat] || 0.0015;
const expectedGainRate = scoreDiff * coef;
```

这会让：
- active/index 的换仓建议更积极（真实预测力强）
- bond/qdii 的换仓建议更保守（预测力弱，不该鼓励换仓）

### Option B：保留全局 0.0013（最小改动）

仅系数微调，不区分类别：

```js
const expectedGainRate = scoreDiff * 0.0013; // 旧 0.0015
```

**Option A 更精准，但改动多 1 处逻辑**。

---

## 🤔 为什么不立即改？

1. **现状系数 0.0015 已经和全样本实证基本一致**（0.89×）
2. **Option A 的分类别系数有风险**：只有 22-60 月数据，类别 R² 差异大可能是采样偏差
3. **优先级**：这个改动改动小、影响小（仅影响 UI 显示的"回本年限"数字和"worthIt" 判定）
4. **配合 3.3 的 `worthIt < 1.5`**：已经变保守了，分类别系数的边际价值进一步递减

## 📋 对算法水平评估的补充

**新信息**：
- scoreF 对 active/index **真的有预测力**（粗分层面）
- scoreF 对 bond/qdii **几乎无预测力**（可能需要专属评分）
- 当前 calculateRebalanceCost 全类别一刀切用 0.0015 — 对 bond/qdii 过度估计 scoreF 价值

**对算法总分的影响**：
- 选基层（用 composite）：不受影响
- 换仓建议层：对 bond/qdii 建议可能略激进
- 算法整体水平：**维持 88-90%**，发现的是局部而非全局问题
